Sobre o Amigo Fiel

Conectando pessoas, ONGs e empresas para adoções responsáveis e bem-estar animal.

O que é

Amigo Fiel é uma plataforma que aproxima adotantes, ONGs e parceiros do mercado pet em um único lugar. Nosso foco é facilitar adoções responsáveis, dar visibilidade a animais resgatados e fortalecer a rede de proteção animal.

Como funciona

• Adotantes encontram animais por espécie, idade e cidade.

• ONGs cadastram pets, gerenciam perfis e recebem contato.

• Empresas divulgam produtos/serviços para o ecossistema pet.

Nossos valores

• Adoção responsável e bem-estar animal.

• Transparência com ONGs e parceiros.

• Comunidade e educação.

Tecnologia

• Django + PostgreSQL.

• Layout responsivo com pages.css.

• Organização por perfis: Comum, ONG e Empresarial.

Tecnologias & Infraestrutura

Resumo da pilha tecnológica utilizada no projeto:

  • Linguagem: Python 3.13
  • Framework: Django 5.2.x
  • Banco de dados: PostgreSQL (psycopg)
  • Templates: Django Templates (server-side rendering)
  • Media/Uploads: armazenados em disco no diretório media/; em produção recomendamos usar um backend S3 compatível
  • Enviar vídeos/embeds: incorporamos YouTube via modo youtube-nocookie por privacidade

Desenvolvimento local

Para rodar localmente:

  1. Crie um ambiente virtual: python -m venv .venv e ative-o.
  2. Instale dependências: pip install -r requirements.txt.
  3. Configure variáveis de ambiente (ex.: POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_HOST, POSTGRES_PORT). Para desenvolvimento com PostgreSQL sem TLS, use POSTGRES_SSLMODE=disable ou deixe o projeto aplicar o padrão DEBUG.
  4. Execute migrações: python manage.py migrate e rode o servidor: python manage.py runserver.

Produção & segurança

  • Habilite TLS/SSL no banco de dados e use POSTGRES_SSLMODE=require em produção.
  • Nunca exponha o SECRET_KEY em repositórios públicos; use variáveis de ambiente seguras.
  • Use um servidor de arquivos (S3, Google Cloud Storage) para grandes volumes de uploads e configure MEDIA_URL/MEDIA_ROOT adequadamente.
  • Considere configurar caching (Redis) e um servidor de aplicação (gunicorn/uvicorn) por trás de um proxy reverso (Nginx) para melhorar performance.

Mais detalhes e instruções de deploy estão disponíveis no repositório do projeto e no README.md.
